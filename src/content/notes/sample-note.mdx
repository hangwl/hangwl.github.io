## Frontmatter Fields

Available frontmatter fields for notes:

- **title** (required): The note title displayed in the UI
- **hide** (optional): Set to `true` to hide from note listings
- **links** (optional): Array of related note slugs (without `.md` extension)

Example:
```yaml
---
title: "My Note Title"
links:
  - 'http'
  - 'https'
  - 'rest-api'
---
```

## Note Writing Guidelines

### Keep It Simple

- **Focused scope**: Each note should cover a single concept or topic
- **Limited sections**: Aim for 3-7 main sections maximum
- **Concise content**: Get to the point quickly, avoid verbose explanations

### When to Create New Notes

If a concept requires extensive explanation:
- Create a separate note for that concept
- Link to it using the `links` frontmatter field
- Reference it inline with markdown links: `[concept name](/notes/concept-slug)`

Example: Instead of explaining SSL/TLS in detail within an HTTPS note, create separate `ssl-tls.md` and `ssl-certificates.md` notes.

### Content Structure

**Good structure:**
```markdown
## Introduction
Brief overview (2-3 sentences)

## Key Concept 1
Concise explanation with examples

## Key Concept 2
Concise explanation with examples

## Best Practices
Bullet points of practical tips
```

**Avoid:**
- Long paragraphs (break into smaller sections)
- Too many subsections (creates cluttered TOC)
- Redundant information (link to related notes instead)
- Tutorial-style step-by-step guides (keep high-level)

### Code Examples

Use code blocks with language specifiers for syntax highlighting:

```python title="Example Code"
def greet(name: str) -> str:
    return f"Hello, {name}!"

print(greet("World"))
```

For plain text or output, omit the language specifier:

```
Output:
Hello, World!
```

### Tables

Use tables for comparisons:

| Feature | Option A | Option B |
|---------|----------|----------|
| Speed | Fast | Slow |
| Cost | High | Low |

### Links

- **Internal notes**: Use `/notes/slug` format
- **External resources**: Use full URLs
- **Related notes**: Add to `links` frontmatter for automatic linking
