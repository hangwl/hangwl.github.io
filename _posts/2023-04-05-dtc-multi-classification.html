Multi-classification using Decision Trees¶

Multi-classification is a common task in machine learning, where we need to classify an instance into one of several possible classes. Decision trees are a popular method for solving multi-classification problems, thanks to their simplicity, interpretability, and ability to handle both numerical and categorical data.

In this exercise, we will be using the built-in sklearn wine dataset. This dataset is commonly used for classification tasks, particularly for multi-class classification using decision trees, logistic regression, and support vector machines.

The load_wine dataset in scikit-learn is a standard machine learning dataset that contains the results of a chemical analysis of wines grown in the same region in Italy, but derived from three different cultivars. The dataset includes 178 wine samples, each with 13 chemical measurements, as well as a target variable indicating the cultivar (i.e., class) of each wine.

The 13 features (i.e., chemical measurements) in the dataset are:

  • Alcohol: the percentage of alcohol present in each wine sample
  • Malic acid: the amount of malic acid present in each wine sample
  • Ash: the amount of ash present in each wine sample
  • Alcalinity of ash: the level of alcalinity of ash present in each wine sample
  • Magnesium: the amount of magnesium present in each wine sample
  • Total phenols: the total amount of phenols present in each wine sample
  • Flavanoids: the total amount of flavanoids present in each wine sample
  • Nonflavanoid phenols: the total amount of nonflavanoid phenols present in each wine sample
  • Proanthocyanins: the total amount of proanthocyanins present in each wine sample
  • Color intensity: the color intensity of each wine sample
  • Hue: the hue (i.e., shade) of each wine sample
  • OD280/OD315 of diluted wines: the ratio of OD280 to OD315 of each wine sample
  • Proline: the amount of proline present in each wine sample
  • The target variable is a categorical variable indicating the cultivar (i.e., class) of each wine sample, with three possible values:
  • Class 0: Wines produced from the first cultivar
  • Class 1: Wines produced from the second cultivar
  • Class 2: Wines produced from the third cultivar

Evaluation Metrics¶

  • Accuracy - Proportion of correctly classified instances out of the total number of instances. Useful to evaluate overall model performance.
  • Precision - Proportion of true positive predictions among all positive predictions. Useful to evaluate the accuracy of the positive predictions.
  • Recall - Proportion of true positive predictions among all actual positive instances. Useful to evaluate the sensitivity of the model in identifying positive instances.
  • F1-score - Harmonic mean of precision and recall. Useful to evaluate the balance between precision and recall.
  • Confusion Matrix - A table that shows the true and predicted class labels. Useful to visualize the types of errors made by the model.

ROC curve and AUC¶

The Receiver Operating Characteristic (ROC) curve is a graphical representation of the performance of a binary classifier as the discrimination threshold is varied. It is created by plotting the True Positive Rate (TPR) against the False Positive Rate (FPR) at various threshold settings.

The ROC curve shows how well the classifier can discriminate between positive and negative instances, and it is particularly useful when the class distribution is imbalanced. A good classifier will have an ROC curve that is closer to the top-left corner of the plot, which corresponds to a high TPR and a low FPR.

The Area Under the ROC Curve (AUC) is a scalar value that represents the overall performance of a binary classifier, independent of the threshold setting. The AUC is equal to the probability that a randomly chosen positive instance is ranked higher than a randomly chosen negative instance by the classifier. An AUC of 1.0 indicates a perfect classifier, while an AUC of 0.5 indicates a random classifier. An AUC between 0.5 and 1.0 indicates a better than random classifier. The AUC is useful for comparing the performance of different classifiers, especially when the class distribution is imbalanced or when the cost of false positives and false negatives is different.
